# Review Assistance Playbook

## Overview

This playbook provides guidelines for peer review in the Cooperation Toolkit. Reviews ensure quality, knowledge sharing, and proper COOK attribution.

## Review Process

### When to Review

- All tasks with COOK value > 0 require review before COOK issuance
- Review is required before task can move to "Done" state
- Multiple reviewers may be required for high-COOK tasks

### Review Checklist

#### Code/Implementation Reviews

- [ ] Code follows team style guidelines
- [ ] Tests are included and passing
- [ ] Documentation is updated
- [ ] No security vulnerabilities
- [ ] Performance considerations addressed
- [ ] Error handling is appropriate

#### Documentation Reviews

- [ ] Content is clear and accurate
- [ ] Formatting is consistent
- [ ] Examples are provided where needed
- [ ] Links are valid
- [ ] Grammar and spelling are correct

#### Process/Policy Reviews

- [ ] Process is clearly defined
- [ ] Policy aligns with team values
- [ ] Implementation is feasible
- [ ] Impact is assessed
- [ ] Stakeholders are considered

## Review Comments

### Effective Review Comments

- **Specific**: Point to exact issues or concerns
- **Constructive**: Suggest improvements, not just problems
- **Respectful**: Maintain professional tone
- **Actionable**: Provide clear guidance on how to address

**Examples:**
- ✅ "Consider adding error handling for the case when the API returns null"
- ✅ "This function could be simplified by extracting the validation logic"
- ❌ "This is wrong"
- ❌ "Fix this"

### Objection Guidelines

- **When to object**: If the work doesn't meet acceptance criteria or quality standards
- **How to object**: Provide clear reasoning and suggest alternatives
- **Resolution**: Work with contributor to resolve objections

## Review Approval

- All required reviewers must approve before task can be completed
- Approval indicates implicit consent for governance (governance-by-workflow)
- Reviews are logged for audit trail

## Best Practices

1. **Review promptly**: Don't let reviews sit for days
2. **Be thorough**: Check all aspects of the work
3. **Ask questions**: If something is unclear, ask for clarification
4. **Provide feedback**: Help contributors improve
5. **Approve when ready**: Don't hold up progress unnecessarily
